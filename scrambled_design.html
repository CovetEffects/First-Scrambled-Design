Lesson One: The Basics of the Web and HTML

What is the World Wide Web? (quick facts)

HMTL - Hyper Text Markup Language

  • HTML was invented in the 1990's, however, it was used as a form of communication between scientists years before it became public to the world.
  • It is the main type of document for the web.
  • When a user goes to a website like facebook.com, he or she is sending a HTTP request to the Internet, then to Facebook's servers. Facebook then sends the request back to the Internet, then the HTML file is interpreted to that specific browser for the user to see.

Different Types of HTML Markup

There are several types of markup within HTML. Essentially, each type of markup is referred to as a type of "tag" (tags look like angled brackets using the greater than and less than symbols) and those tags combined with text (content) form an "element".

Elements can be in-line or in block form. Each element will have an opening and closing tag. However, if it does not have a closing tag, it is referred to as a "void" tag.

<tag>Types of Elements</tag>

  • In-line: An element that does not force new lines before or after its placement, i.e., <em>, <a> (anchor tag), <br> (line breaking tag. < div> can also be an in-line tag because of the span element.
  • Block: An element has lines that appear before and after the element, i.e, <p>, <h1>, <div>, essentially, it is enclosed in a block.

Lesson Three: Adding CSS Style to HTML Structure

CSS (Cascading Style Sheet) is what gives HTML the "style" to the elements within the code by eliminating repetition. There are a few methods of adding CSS to the HTML document. In addition, CSS is a search replacement tool that allows you to identify a class or tag of an element chosen to style and change that element according to preference. With great power, comes great responsibility - there are rules to this power.

Three Methods to Placing the Style Sheet

  1. CSS in Separate File: creating a separate style_sheet.css is the ideal way of controlling CSS. This method allows you to stay more organized and is the ideal method of placing CSS.
  2. CSS in the <head> of the HTML: This method is good for small projects. The CSS is enclosed in the <style> tag nested within the <head> element.
  3. In-line Style: This method is generally not recommended because it leads to a lot of repeated code. For further reading on the debate of in-line CSS, Stackoverflow gives an excellent explanation of the questions and the opinions of other Devs.

Rules of Cascading

  • Cascading: defined as rules to be applied not only to the elements they directly match, but also to all those elements' child elements. However, if a child element has multiple overlapping rules defined for it, the more specific rules takes effect.
  • Inheritance:
    • relies on the ancestor - descendant relationship. It applies all changes to the specific element and all it's descendants.
    • Adheres to the document tree/heirarchy of XHTML, based off of nesting
    • Enhances faster loading speed of web-pages by the user.
    • Helps the client to save money on bandwidth and Dev costs.

Understanding Selectors

Not that we understand the methods and rules of CSS, how do the elements get changed by the CSS. CSS has the ability to "select" all elements of the same kind and change the value by calling a specific attribute. As a result, you are making a new set of "rules" within your main.css file.

CSS control over HTML

  • h1 {
    background-color: blue;
    color: white;
    }
    • h1 = Selector
    • background-color = attribute
    • blue = value
    • { } = everything within the brackets is now a rule

So what is this actually doing to the HTML document? Basically, the CSS is selecting all elements within the <h1> tag and telling them they are now going to be with a blue background and with white text.

Selecting by Class

Now that we know how to change the style via CSS page, we can also do this by using class selectors, i.e., <div>.

  • <div class="lesson_one">Content for Div</div>
  • .lesson_one {
    background-color: blue;
    color: white;
    }

In the above example, we identified the class as "lesson_one" in the main.html file and within the main.css file we styled that class with a background of blue and a text color of white.

Selecting by ID

Another form of selection is by selecting by an ID selector. ID selectors are the most powerful type of selector in terms of CSS specificity. As a result, ID selectors beat out other types of selectors and the styles defined within, therefore can't be altered. On the other hand, having lower-specificity selectors that are easier to override is a plus, i.e., class selectors.

  • CSS Code:
    • #firstname {
      background-color: yellow;
      color: white;
      }
      #future_class {
      background-color: black;
      color: white;
      }
  • HTML Code:
    • <p id="firstname">My name is Kevin!</p>
      <p id="future_class">I want to learn Data Science!</p>

Boxes, Boxes, and more Boxes!

Box Positioning

If you were to open the developer tools in either Firefox, Google Chromes, or Internet Explorer, you will notice the web-page is made up of boxes, which are containers or divs. These divs or block elements take up the entire width of the page. However, by adding the display: flex; rule to the CSS to the corresponding area will allow divs be next to one another.

Resizing Boxes

Resizing boxes seemed to be a pretty fun and exploring part of learning how CSS and HTML interact with one another. In order to change the way a box is visible on the HTML document, there are four different components to adjust the size of the box: margin, border, padding, and content.

CODE, TEST, REFINE

As a complete beginner to learning how HTML and CSS work, it is important to follow a set of guidelines to stay focused and organized.

CODE

  • Deciding what the overall design mock will be (boxing), deciding the structure.
  • Writing the code for HTML and CSS.
  • CSS - make the style changes beginning with biggest to smallest

TEST

  • Testing the code on other browsers.
  • Is the color and text correct? Are the boxes in the correct order/layout?

REFINE

  • Fix all desired changes
  • Continue testing and refining till the web-page looks like the mock-up.

Developer Tools

Developer tools, that is all I can say, developer tools. It is one of the best tools in the arsenal of a developer. Why? You can look at each element and tweak it to the desired look without changing any code on your original document. No more saving the code to the HTML document and refreshing your browser. When you are done, you can make the desired changes to your original document.

With developer tools, you are able to:

  • Change or add elements
  • Change CSS, i.e., color, text, etc.
  • Rearrange the layout of the document.
  • Resize boxes and positioning

Verifying HTML and CSS

Verifying the HTML and CSS can be part of the testing and refining process. You will need to visit W3C Markup Validation Service to check your HTML. After verifying your HTML, you will also have to verify your CSS at W3C CSS Validation Service. If there are any errors, those errors will be displayed and with helpful links and instructions on how to correct the code so it passes verification.
